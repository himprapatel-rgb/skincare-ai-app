## 1. Design System

### 1.1 Color Palette

Based on top skincare app analysis:

```
PRIMARY COLORS:
- Primary Blue:     #4A90E2 (Trust, Medical)
- Soft Pink:        #FFB6C1 (Gentle, Skincare)
- Mint Green:       #98D8C8 (Fresh, Natural)
- Warm Peach:       #FFDAB9 (Warmth, Care)

SECONDARY COLORS:
- Deep Navy:        #1A365D (Headers)
- Light Gray:       #F7FAFC (Backgrounds)
- Medium Gray:      #E2E8F0 (Borders)
- Dark Text:        #2D3748 (Body Text)

- Success Green:    #48BB78
- Warning Yellow:   #ECC94B
- Error Red:        #F56565
- Info Blue:        #4299E1

- Body Regular:         14px / Regular

```

## 12. Summary & Design Principles

### 12.1 Key Design Decisions

1. **Bottom Navigation:** 5 primary tabs for quick access to main features
2. **Card-Based UI:** Content grouped in cards for visual hierarchy
3. **Consistent Icons:** Emoji/SF Symbols throughout for recognition
4. **Progress Indicators:** Clear feedback during ML processing
5. **Data Visualization:** Charts and graphs for skin progress tracking

### 12.2 Technology Integration Points

| Feature | Technology | Platform |
|---------|------------|----------|
| Camera | flutter_camera | Flutter |
| Face Detection | Google ML Kit | TFLite |
| Skin Analysis | Custom PyTorch â†’ ONNX â†’ TFLite | On-device |
| Barcode Scan | ML Kit Barcode | Android/iOS |
| OCR | ML Kit Text Recognition | Android/iOS |
| Notifications | flutter_local_notifications | Flutter |
| Storage | SQLite + Hive | Local |
| API | FastAPI | Backend |
| Database | PostgreSQL | Backend |

### 12.3 Accessibility Considerations

- **High Contrast:** Minimum 4.5:1 contrast ratio for text
- **Touch Targets:** Minimum 44x44pt touch targets
- **Screen Reader:** All images have alt text
- **Font Scaling:** Dynamic type support
- **Color Independence:** Information not conveyed by color alone
- **Motion:** Reduced motion option available

### 12.4 Performance Guidelines

- **Cold Start:** < 3 seconds to interactive
- **ML Analysis:** < 2 seconds on mid-range devices
- **Navigation:** < 100ms screen transitions
- **Images:** Lazy loading with placeholders
- **Offline:** Core features work offline

### 13.5 Accessibility Checklist (WCAG 2.1 AA)

#### Typography
- [ ] Minimum font size: 16sp (body text)
- [ ] Line height: 1.5x font size minimum
- [ ] Font scaling: Support up to 200% zoom
- [ ] Dyslexia-friendly: Left-aligned text, avoid justified

#### Color & Contrast
- [ ] Text contrast ratio: 4.5:1 minimum (normal text)
- [ ] Large text contrast: 3:1 minimum (18sp+ or 14sp bold)
- [ ] No color-only information (add icons/text)
- [ ] Dark mode support with proper contrast

#### Touch Targets
- [ ] Minimum target size: 44x44 dp (iOS) / 48x48 dp (Android)
- [ ] Spacing between targets: 8dp minimum
- [ ] Touch feedback: Visual ripple or highlight

#### Screen Reader Support
- [ ] All images have `contentDescription` / `accessibilityLabel`
- [ ] Form fields have labels
- [ ] Buttons have descriptive labels
- [ ] Focus order is logical

#### Motion & Animation
- [ ] Respect `Reduce Motion` system setting
- [ ] No auto-playing videos without controls
- [ ] Animations can be paused/stopped
- [ ] No content that flashes >3 times/second

#### Navigation
- [ ] Skip navigation option for screen readers
- [ ] Consistent navigation patterns
- [ ] Current location always indicated
- [ ] Back button always functional
